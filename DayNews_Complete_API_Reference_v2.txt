1. Introduction
This document provides complete API reference documentation for the Day.News platform, covering all 164+ database tables organized into 28 functional categories. The API follows RESTful conventions and uses JSON for request/response payloads.

1.1 Base URL
Production: https://api.day.news/v1
Staging: https://api-staging.day.news/v1

1.2 Authentication
All API requests require authentication via Bearer token in the Authorization header:
Authorization: Bearer {access_token}
Tokens are obtained through the /auth/login endpoint and expire after 24 hours.

1.3 Response Format
All responses follow a consistent JSON structure:
{ "success": true, "data": {...}, "meta": { "page": 1, "per_page": 20, "total": 100 } }

1.4 Error Handling
Errors return appropriate HTTP status codes with descriptive messages:
{ "success": false, "error": { "code": "VALIDATION_ERROR", "message": "...", "details": [...] } }

Appendix A: HTTP Status Codes
Standard HTTP status codes used by the API:
	•	200 OK — Successful request
	•	201 Created — Resource created successfully
	•	204 No Content — Successful deletion
	•	400 Bad Request — Invalid request data
	•	401 Unauthorized — Authentication required
	•	403 Forbidden — Insufficient permissions
	•	404 Not Found — Resource not found
	•	422 Unprocessable Entity — Validation error
	•	429 Too Many Requests — Rate limit exceeded
	•	500 Internal Server Error — Server error

Appendix B: Rate Limits
API rate limits vary by endpoint and authentication level:
	•	Unauthenticated: 60 requests/minute
	•	Authenticated: 120 requests/minute
	•	Premium: 300 requests/minute
	•	Enterprise: Custom limits

Appendix C: Pagination
List endpoints support cursor-based pagination:
GET /posts?page=1&per_page=20&sort=created_at&order=desc
Response includes pagination metadata in the 'meta' object.
